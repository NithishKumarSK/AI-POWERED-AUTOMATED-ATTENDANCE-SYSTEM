            1.2  EXISTING SYSTEM 
	The Existing System comprises basic online examination platforms that primarily support objective-based assessments, such as multiple-choice and true/false questions, with automated grading. While automation speeds up evaluation, it lacks accuracy in assessing complex responses, leading to unfair scoring. Some platforms offer webcam monitoring and tab-switch detection, but they lack real-time analysis, making them vulnerable to cheating and impersonation. User authentication is often limited to passwords or OTPs, with minimal identity verification. Most systems do not employ facial recognition, allowing unauthorized individuals to take exams on behalf of others. Additionally, they fail to detect multiple faces in the frame, making impersonation easier. There is no effective monitoring of cell phone usage or other prohibited objects, allowing students to access unauthorized materials without detection. Many platforms also struggle to track tab-switching accurately, enabling candidates to browse external resources during the exam. The absence of real-time alerts for suspicious activities further weakens security, requiring manual review of recorded sessions. Moreover, some systems do not enforce continuous face detection, allowing students to leave the screen undetected. Due to these limitations, existing systems fail to ensure fairness, efficiency, and security, highlighting the need for advanced AI-driven proctoring solutions.
 PROPOSED SYSTEM 
The Proposed System introduces an advanced AI-driven examination platform designed to enhance security, fairness, and efficiency in online assessments. It integrates real-time facial recognition using OpenCV to verify identity and prevent impersonation, while multiple face detection ensures only the registered candidate is present. Cell phone and prohibited object detection using YOLO with TensorFlow prevents unauthorized material usage, and continuous face 
detection ensures students remain visible throughout the exam. Tab-switch monitoring using JavaScript browser APIs restricts access to external resources, maintaining a controlled test environment. Additionally, intelligent automated grading optimizes evaluation accuracy, reducing manual effort and ensuring fair, consistent scoring. Real-time alerts notify administrators of suspicious activities, eliminating the need for extensive manual review. Built with React.js for the frontend, Node.js with Express.js for the backend, and MongoDB for secure data storage, the system ensures scalability, seamless integration, and high performance. By combining automated evaluation with intelligent proctoring, the proposed system sets a new benchmark for secure, efficient, and technologically advanced online assessments.

Live monitoring via webcam

Step 2: Technology Stack
Frontend: React.js / Angular / Vue.js (for exam interface)

Database: PostgreSQL / MySQL / MongoDB (to store user data, exam 
  results)
AI/ML Libraries: OpenCV, TensorFlow/Keras, Scikit-learn, NLP tools

3. Smart Proctoring Implementation
(a) Face & Voice Detection
Use OpenCV & Dlib for real-time face tracking.
Use Deep Learning (CNNs) to detect multiple faces.
Raise flags for suspicious activity (e.g., if another person appears).
(b) Browser Activity Monitoring
Restrict tab switching using JavaScript event listeners.
Log key events (like copy-pasting, tab switching, minimizing the browser).
Detect external devices using WebRTC APIs.
(c) Cheating Detection
Use Gaze Tracking (eye movement) to detect if a student looks away too Often.
Implement Object Detection (YOLO, Mobile Net) to identify unauthorized  devices.
Record suspicious activities and generate a cheating probability score.

Relevance (using NLP similarity metrics).

Flags for suspicious activities.
Scores for each question.
AI-generated feedback for improvement.

import mongoose from "mongoose";
const connectDB = async () => {
  try {
    if (!process.env.MONGO_URL) {
      throw new Error("MONGO_URL is not defined. Please check your .env file.");
    }
    console.log(`🔄 Connecting to MongoDB at: ${process.env.MONGO_URL}`);
    const conn = await mongoose.connect(process.env.MONGO_URL, {
      useNewUrlParser: true,
      useUnifiedTopology: true,
});
    console.log(`✅ MongoDB Connected: ${conn.connection.host}`);
  } catch (error) {
    console.error(`❌ Database Connection Error: ${error.message}`);
    // Optional: Retry connection after a delay
    // setTimeout(connectDB, 5000); // Uncomment to retry every 5 seconds
    process.exit(1); // Exit process if connection fails
  }
};
export default connectDB;
